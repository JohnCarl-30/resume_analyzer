## Priority Issues

### P0 — analysis-workspace.tsx God Component (1,272 lines, 15+ useState)
1,272 lines with 4 modals, 5 editor branches, autosave, undo/redo, keyboard shortcuts, mobile/desktop layout. Impossible to test. Extract into useWorkspaceModals, extend useResumeEditor, split renderEditor into WorkspaceSidebar. Target: each piece <300 lines.
Fix: $impeccable shape

### P1 — Keyboard Shortcuts Modal Missing Dialog Primitives
Uses raw `fixed inset-0 z-30` instead of shadcn Dialog. No focus trap, no Escape handling, no aria-modal. Screen readers won't announce it.
Fix: $impeccable harden

### P1 — No Unsaved Changes Guard on Back Navigation
onBack called directly. No beforeunload or confirmation dialog when createMode && draftStatus === 'unsaved'. Users could lose work.
Fix: $impeccable harden

### P2 — Content Modal Overwhelms with 7 Ungrouped Options
Flat grid, no categorization. Users see >4 choices with no grouping. Leverage analysis results to show missing/weak sections first.
Fix: $impeccable distill

### P2 — Workspace Header Has 8+ Visible Controls
Back, title, score, undo/redo, status, review, check again, template, print, download, shortcuts, account — all visible. Collapse low-frequency into overflow.
Fix: $impeccable distill
